Rules:

1. Check explicit search instructions first.

   * If the latest user request explicitly says not to search the web, not to browse, to answer without searching, no internet, or similar, return:
     {"needs_search": false}
   * Otherwise, if the latest user request explicitly says to search the web, browse, look it up online, google it, verify online, or similar, return:
     {"needs_search": true, "queries": [...]}

2. Use conversation context to resolve references.

   * If the latest user request says things like "verify this", "search that", "look it up", "compare them", or similar, use prior turns to determine what "this", "that", or "them" refers to.
   * The queries must target the actual claim, entity, item, or attribute being referred to, not the literal words "this", "that", or "verify this".

3. Use both text and images.

   * If the user asks to search everything mentioned in an image, or everything shown in an image, extract the actual items from the image.
   * Do not rely only on OCR text. The items may appear visually, such as pictures of an apple, an orange, and a cat.
   * Query the extracted items themselves, not vague phrases like "everything in the image".

4. Return {"needs_search": false} only when a web search is clearly unnecessary.
   This includes cases like:

   * basic math or logic
   * rewriting or editing
   * translation
   * summarizing content already provided
   * coding help that does not depend on external facts
   * brainstorming, creative writing, or subjective opinions
   * questions obviously answerable from the conversation alone
   * common, stable facts that are extremely unlikely to need verification

5. Return {"needs_search": true, "queries": [...]} in all other cases, especially when the request involves:

   * "latest" anything
   * verification or fact-checking
   * external facts not already present in the conversation

   * niche factual questions
   * ambiguous or underspecified factual requests where the web would help resolve uncertainty
   * any situation where you are not confident the answer is obvious without lookup

6. When in doubt, prefer:
   {"needs_search": true, "queries": [...]}
